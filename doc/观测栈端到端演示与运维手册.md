# FinQuanta 观测栈端到端演示与运维手册

日期：2026-04-10  
适用范围：`Grafana + Tempo + Loki + Promtail + FinQuanta API + OpenClaw Gateway`

---

## 1. 目标

本手册用于完成两件事：

1. 给评审/验收做一段可复现的端到端观测演示录屏（trace/alerts/logs 三联）。
2. 给运维和联调同学提供一份可直接执行的排障手册。

---

## 2. 环境准备

### 2.1 必要组件

- Python 3.10+
- Docker Desktop（可用 `docker compose`）
- Node.js（用于 OpenClaw Gateway）
- 可访问本机端口：`9000`、`18789`、`3000`、`3100`、`3200`、`4318`

### 2.2 关键配置（建议）

在 `.env.api.local` 或 `.env.api` 中确认：

```env
FINQUANTA_API_BASE=http://127.0.0.1:9000
FINQUANTA_OPENCLAW_GATEWAY_ENABLED=1
FINQUANTA_OPENCLAW_GATEWAY_BASE=http://127.0.0.1:18789
FINQUANTA_OBSERVABILITY_READ_TOKEN=finquanta_obs_demo_token
```

---

## 3. 启动顺序（推荐）

### Step 1: 启动 OpenClaw Gateway

```bat
start_openclaw.bat
```

仅探活：

```bat
start_openclaw.bat --health-only
```

### Step 2: 启动 API

```bat
start_api.bat
```

### Step 3: 启动观测栈

```bash
python infra/oneclick_observability_stack.py --api-base http://127.0.0.1:9000 --overwrite --up
```

### Step 4: 健康检查

```bash
python infra/check_observability_stack.py
python infra/check_openclaw_gateway.py --strict
```

---

## 4. 录屏脚本（建议 8-12 分钟）

## 4.1 开场（约 1 分钟）

展示目标：

- API 在 `9000`
- Gateway 在 `18789`
- Grafana 在 `3000`
- 目标是看到 trace/alerts/logs 三联联动

展示命令输出：

```bat
start_openclaw.bat --health-only
python infra/check_observability_stack.py
```

## 4.2 触发业务流量（约 2-3 分钟）

运行轻量联调：

```bat
smoke_api.bat relaxed
```

说明：

- `relaxed` 模式用于演示时避免因为网关临时波动中断全流程。
- CI 建议用 `strict`。

## 4.3 进入 Grafana 演示三联（约 3-5 分钟）

1. 打开三联示例面板：`infra/dashboard_examples/grafana_trace_alerts_logs_trilink_dashboard.json`（已 provisioning 后可直接看到）。
2. 切到 Trace 视图，展示最近链路。
3. 切到 Alerts 视图，展示策略评估和路由执行状态。
4. 切到 Logs 视图，筛选 `structured.log` 采集日志并按 `trace_id` 关联。
5. 演示 panel-input 聚合接口：

```text
GET /api/observability/dashboard/panel-input?obs_token=<token>
```

## 4.4 收尾（约 1-2 分钟）

展示验收结论：

- 链路有 trace
- 告警有策略与回执
- 日志可按 trace_id 关联检索
- 观测链路可一键启动、可健康检查、可回归

## 6. 常见故障排查

## 6.1 Gateway 不可达

现象：

- `check_openclaw_gateway.py --strict` 报 `[FAIL] OpenClaw gateway is unreachable`

处理：

1. 执行 `start_openclaw.bat` 启动网关。
2. 检查 `FINQUANTA_OPENCLAW_GATEWAY_BASE` 是否与启动端口一致。
3. 如果机器上已有旧实例占用端口，先释放端口再重启。

---

## 6.2 API 可用但 OpenClaw 调用走本地回退

现象：

- `/api/openclaw/pipeline/run` 结果中 `gateway.mode=fallback_local`

处理：

1. 检查 `FINQUANTA_OPENCLAW_GATEWAY_ENABLED=1`
2. 检查 `FINQUANTA_OPENCLAW_GATEWAY_BASE` 与网关地址一致
3. 检查 `FINQUANTA_OPENCLAW_GATEWAY_PIPELINE_PATHS`、`FINQUANTA_OPENCLAW_GATEWAY_LEARN_PATHS` 是否含正确路径

---

## 6.3 Grafana 无数据

现象：

- 面板为空，Trace/Logs 均无数据

处理：

1. 先跑 `smoke_api.bat relaxed` 触发流量
2. 检查 `python infra/check_observability_stack.py`
3. 确认 `infra/oneclick_observability_stack.py --up` 成功
4. 检查 Grafana datasource 是否存在 `FinQuanta Tempo`、`FinQuanta Loki`

---

## 6.4 Loki 有数据但日志不关联 trace

现象：

- logs 有内容，但看板按 trace_id 过滤不到

处理：

1. 确认结构化日志路径为 `FINQUANTA_STRUCTURED_LOG_PATH`（默认 `logs/observability/structured.log`）
2. 确认 Promtail 已挂载并采集该路径
3. 确认相关链路代码写入了 `trace_id` 字段（结构化日志）

---

## 6.5 smoke_api 严格模式失败

现象：

- `smoke_api.bat strict` 在网关检查阶段退出

处理：

1. 本地开发可切 `smoke_api.bat relaxed`
2. CI 保持 strict，并在作业前显式启动 Gateway

---

## 7. 建议的 CI/本地策略

- 本地开发：`smoke_api.bat relaxed`
- 联调验收：`smoke_api.bat strict`
- CI：`smoke_api.bat strict` + `check_observability_stack.py`

---

## 8. 关联命令速查

```bash
# OpenClaw Gateway 检查
python infra/check_openclaw_gateway.py --strict

# API 冒烟
smoke_api.bat strict
smoke_api.bat relaxed

# 观测栈一键启动/停止
python infra/oneclick_observability_stack.py --api-base http://127.0.0.1:9000 --overwrite --up
python infra/oneclick_observability_stack.py --down

# 观测栈健康检查
python infra/check_observability_stack.py
```
