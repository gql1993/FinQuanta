# FinQuanta 改造前备份与基线清单

下面是一份 **Windows PowerShell 可直接执行的命令级 checklist**，按当前项目 `E:\work\FinQuanta` 的现状整理。

目标是先把“**代码基线 + 配置基线 + 数据基线 + 运行基线**”固化下来，再开始改造。

## 0. 进入项目目录

```powershell
cd E:\work\FinQuanta
```

## 1. 创建基线备份目录

建议把备份放在仓库外，避免再次被 Git 跟踪。

```powershell
$DATE = Get-Date -Format "yyyy-MM-dd_HHmm"
$BACKUP_ROOT = "E:\backup\FinQuanta_baseline_$DATE"
New-Item -ItemType Directory -Force -Path $BACKUP_ROOT
New-Item -ItemType Directory -Force -Path "$BACKUP_ROOT\config"
New-Item -ItemType Directory -Force -Path "$BACKUP_ROOT\data"
New-Item -ItemType Directory -Force -Path "$BACKUP_ROOT\reports"
New-Item -ItemType Directory -Force -Path "$BACKUP_ROOT\git"
```

验证目录是否创建成功：

```powershell
Get-ChildItem $BACKUP_ROOT
```

## 2. 固化 Git 基线

先记录当前状态，再打 tag 和保护分支。

### 2.1 导出当前 Git 信息

```powershell
git status --short --branch | Out-File "$BACKUP_ROOT\git\git-status.txt" -Encoding utf8
git log --oneline -10 | Out-File "$BACKUP_ROOT\git\git-log.txt" -Encoding utf8
git remote -v | Out-File "$BACKUP_ROOT\git\git-remotes.txt" -Encoding utf8
git rev-parse HEAD | Out-File "$BACKUP_ROOT\git\git-head.txt" -Encoding utf8
```

### 2.2 给当前版本打基线 tag

当前已上传的基线提交是 `f771a1e`，建议直接打 tag：

```powershell
git tag baseline/pre-refactor-2026-04-08 f771a1e
```

确认 tag：

```powershell
git tag --list "baseline/*"
```

### 2.3 创建保护分支

```powershell
git branch baseline/current-runnable f771a1e
```

确认分支：

```powershell
git branch --list
```

### 2.4 推送基线 tag 和保护分支到新仓库

如果希望连基线也推到 `finquanta` 远程：

```powershell
git push finquanta baseline/current-runnable
git push finquanta baseline/pre-refactor-2026-04-08
```

## 3. 备份配置文件

先看有哪些 `.env` 文件存在：

```powershell
Get-ChildItem -Force . -Filter ".env*" | Select-Object Name,Length,LastWriteTime
```

把存在的配置文件复制到备份目录：

```powershell
if (Test-Path ".env") { Copy-Item ".env" "$BACKUP_ROOT\config\" -Force }
if (Test-Path ".env.api") { Copy-Item ".env.api" "$BACKUP_ROOT\config\" -Force }
if (Test-Path ".env.api.local") { Copy-Item ".env.api.local" "$BACKUP_ROOT\config\" -Force }
if (Test-Path ".env.api.example") { Copy-Item ".env.api.example" "$BACKUP_ROOT\config\" -Force }
```

再导出一份文件清单：

```powershell
Get-ChildItem "$BACKUP_ROOT\config" | Select-Object Name,Length,LastWriteTime | Out-File "$BACKUP_ROOT\reports\config-files.txt" -Encoding utf8
```

## 4. 备份 SQLite 数据

先确认数据库文件：

```powershell
Get-ChildItem ".\data_cache" -Force
```

如果桌面程序已经关闭，执行数据库备份：

```powershell
if (Test-Path ".\data_cache\quant.db") {
    Copy-Item ".\data_cache\quant.db" "$BACKUP_ROOT\data\quant_before_refactor.db" -Force
}
if (Test-Path ".\data_cache\quant.db-wal") {
    Copy-Item ".\data_cache\quant.db-wal" "$BACKUP_ROOT\data\quant_before_refactor.db-wal" -Force
}
if (Test-Path ".\data_cache\quant.db-shm") {
    Copy-Item ".\data_cache\quant.db-shm" "$BACKUP_ROOT\data\quant_before_refactor.db-shm" -Force
}
```

验证备份结果：

```powershell
Get-ChildItem "$BACKUP_ROOT\data"
```

## 5. 备份关键说明文档

这些文档后面会改，建议先留一份基线副本：

```powershell
Copy-Item ".\README.md" "$BACKUP_ROOT\reports\" -Force
```

把当前启动命令写进文本文件，后续每次对照。

```powershell

桌面启动:

```

## 7. 做最小运行冒烟记录

### 7.1 API 健康检查

如果本地还没启动 API，先启动：

```powershell
python -m uvicorn api_server.main:app --host 127.0.0.1 --port 9000
```

另开一个 PowerShell 窗口，再执行：

```powershell
Invoke-WebRequest "http://127.0.0.1:9000/health" | Select-Object -ExpandProperty Content
Invoke-WebRequest "http://127.0.0.1:9000/health/deps" | Select-Object -ExpandProperty Content
```

把结果写入基线报告：

```powershell
Invoke-WebRequest "http://127.0.0.1:9000/health" | Select-Object -ExpandProperty Content | Out-File "$BACKUP_ROOT\reports\health.json" -Encoding utf8
Invoke-WebRequest "http://127.0.0.1:9000/health/deps" | Select-Object -ExpandProperty Content | Out-File "$BACKUP_ROOT\reports\health-deps.json" -Encoding utf8
```

### 7.2 登录接口验证

```powershell
$body = @{
    username = "admin"
    password = "admin123"
} | ConvertTo-Json

Invoke-WebRequest "http://127.0.0.1:9000/api/auth/login" `
    -Method POST `
    -ContentType "application/json" `
    -Body $body | Select-Object -ExpandProperty Content
```

导出结果：

```powershell
Invoke-WebRequest "http://127.0.0.1:9000/api/auth/login" `
    -Method POST `
    -ContentType "application/json" `
    -Body $body | Select-Object -ExpandProperty Content | Out-File "$BACKUP_ROOT\reports\login.json" -Encoding utf8
```

### 7.3 运行现有冒烟脚本

```powershell
.\check_api.bat
.\smoke_api.bat
```

如果有输出，建议手工复制终端结果，或重定向：

```powershell
cmd /c check_api.bat > "$BACKUP_ROOT\reports\check_api.txt" 2>&1
cmd /c smoke_api.bat > "$BACKUP_ROOT\reports\smoke_api.txt" 2>&1
```

### 7.4 PostgreSQL 验证

如果当前有 Postgres 环境：

```powershell
.\validate_postgres.bat
```

导出结果：

```powershell
cmd /c validate_postgres.bat > "$BACKUP_ROOT\reports\validate_postgres.txt" 2>&1
```


提交这部分基线文档：

```powershell
git add .\doc\baseline
git commit -m "add baseline documentation before refactor"
```
